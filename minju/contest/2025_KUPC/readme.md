### **B. 얼룩말과 사자 (Lion & Zebra)**

- **유형:** 수학, 정수론
- **문제:** 사자가 매년 얼룩말을 잡아먹을 때, 멸종하지 않기 위한 최소 개체 수 구하기.
- **접근:** 시뮬레이션(`while`)으로 접근했으나 시간 초과 우려로 수학적 수식(2N)을 도출함.
- **Trouble Shooting:** - 수식은 정확했으나 입력값(N \le 10^{18})이 JavaScript `Number`(2^{53}-1) 범위를 초과하여 오답 처리됨.
  - **해결:** `BigInt` 자료형을 사용하여 오버플로우 문제 해결.
- **Lesson:** **"입력값이 2^{53}을 넘어가면 무조건 `BigInt`를 사용하자."**

### **C. 코끼리 15마리가 앞으로 돌진하면 어떻게 막을 건데**

- **유형:** 그리디(Greedy), 엣지 케이스 처리
- **문제:** 당근을 먹으면 사라지거나 잠드는 코끼리의 특성을 고려해 돌진을 막아야 함.
- **Trouble Shooting:** - 단순히 "가장 코가 긴 코끼리"(`MAX`)에 맞추면 된다고 생각했으나, "당근을 먹으면 사라진다"는 조건을 간과함.
  - **해결:** 앞쪽 코끼리들을 기절시키기 위한 값(`MAX + 1`)과 마지막 코끼리를 재우는 값 중 최댓값을 찾는 로직으로 수정.
- **Lesson:** **"문제의 특수 조건(Constraint)을 시각화하여 엣지 케이스를 파악하는 것이 우선이다."**

### **D. 같은 배열 (Same Array)**

- **유형:** 자료구조(Set), 애드혹(Ad-hoc)
- **문제:** 배열 A의 원소를 복사해 배열 B를 만들 수 있는지 검증.
- **Trouble Shooting:** - 초기 코드에서 반복문 내 `Array.includes()` 사용으로 인해 시간 복잡도가 O(N^2)로 증가 (N=200,000일 때 시간 초과).
  - **해결:** `Set` 자료구조를 활용하여 탐색 시간을 O(1)로 단축, 전체 로직을 O(N)으로 최적화.
- **Lesson:** **"대량의 데이터 검색 시 `Array` 대신 `Set/Map`을 사용하는 습관을 들이자."**

### **E. 꿈 (Dream)**

- **유형:** 그리디(Greedy), 역방향 탐색
- **문제:** 기억해야 할 날(S)을 피해서 악몽으로 모든 잡초(잊어야 할 날)를 지우기.
- **Trouble Shooting:** - 1일부터 N일까지 순차적으로 탐색하며 지우려 했으나, 미래의 악몽이 과거에 영향을 주는 경우를 처리하기 어려웠음.
  - **해결:** N일부터 1일까지 **역방향(N \to 1)**으로 탐색하며, 잡초가 보이면 즉시 악몽을 꾸는 방식으로 해결.
- **Lesson:** **"순방향으로 풀리지 않는 그리디 문제는 역방향 탐색이 열쇠일 수 있다."**
